تقریبا همه توسعه‌دهندگان css از flex برای پیاده‌سازی چیدمان‌های متفاوت استفاده می‌کنند. اما موقعی که از flex استفاده می‌کنیم، ممکن است عرض المان پدر برای جاگیری همه المان‌های فرزند در یک خط کم باشد. در این صورت المان‌های فرزند از پدر خود بیرون میفتند. برای جلوگیری از این مشکل می‌توان از flex-wrap: wrap; استفاده کرد.

03 >Ata! DEFENSIVE CSS

در دنیای پروانه‌ها و رنگین کمان، متن‌ها همگی به اندازه هستند و همگی در المانی که تعبیه شده‌اند به خوبی جا می‌گیرند و از اون بیرون نمی‌زنند. اما در دنیای واقعی، ما باید آماده هر نوع متن با هر اندازه‌ای باشیم. برای این بهتره تا حدی که المان پدر اجازه میده تا متن بدون اینکه بشکنه یا ازش بزنه بیرون، متن رو نشون بدیم و بقیه‌اش رو سه نقطه قرار بدیم که با دستور text-overflow: ellipsis; قابل دستیابیه.

HOVER ON MOBILE DEVICES

06 >Ata! DEFENSIVE CSS

وقتی برای hover میایم یه سری styleها رو تعریف می‌کنیم، مخاطبمون از این کارها کسایی هست که از mouse یا trackpad استفاده می‌کنن. با موبایل هم میشه حالت hover رو با نیمه لمس کردن فعال کرد اما نه تنها این فعال کردن خوب نیست بلکه میتونه به کاربر تجربه بدی بده. ما میتونیم با @mdia (hover: hover){ // code } این مشکل رو برطرف کنیم.

HIDDEN SCROLLBARS

07 >Alireza DEFENSIVE CSS

برای بعضی آیتم‌ها که محتوای طولانی دارند بهتره scrollbar قرار بدیم. اما ممکنه متن ما متغیر باشه، و همیشه scrollbar به کارمون نیاد و در این حالت scrollbar نباید نشون داده بشه. اگر از دستور overflow-y: auto; استفاده کنیم، فقط در صورت نیاز scrollbar نشون داده میشه.

STICKY GRIDS

08 >Alireza DEFENSIVE CSS

ممکنه وقتی کل چینش سایت رو با grid انجام دادین، نیاز شده باشه که قسمتی رو sticky کنید، مثل منوی کناری. در این حالت خواهید دید که اون المان تمام ارتفاع کلی grid رو می‌گیره.
در این حالت باید با align-self: start; اون المان رو از حالت پیش فرض stretch در بیارین و به start تغییرش بدین.
